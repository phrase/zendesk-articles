There are four steps to publishing Release Notes:

  1. Create the RN

  2. Send the RN for review

  3. Publish the RN

  4. Trigger notifications

To create a Release Note article, follow these steps:

  1. Confirm release note version number, product, release date and associated issue numbers with product owner.

  2. Open the Release Notes folder in Paligo and expand the folder for the specific release (TWE, Mobile, etc.).

  3. Left click the context menu on the right of the release folder and select Create content.

    The Create content window opens.

  4. For document name, provide version number and release date based on previous releases.

    Copy this name.

  5. Click From template and select Release Note.

    Release Note is indicated as the template.

  6. Select Open in editor and Click Ok.

    The new article is opened in the editor.

  7. Replace Date & Version with text copied in step 4.

  8. Add Features and/or Bugs to the article.

  9. Add links to HC articles where appropriate.

  10. Save the article.

To send the RN for review, follow these steps:

  1. Click the context menu for the article and select Create assignment and Review.

    The Review assignment page opens.

  2. Select an Assignee from the drop down list. This is typically the production owner.

  3. Select an Assignment date. Only the end date is required and is typically the Thursday before the release.

  4. Optionally select Let me leave a message if there is something the review should know. This message is placed in the notification email sent to the reviewer.

  5. Click Ok.

    A notification email is sent to the specified Assignee containing a link to the article in a Paligo review page. A notification email is also sent to the editor.

    When the reviewer marks the article as complete or rejected (requiring modification), a second notification email is sent to the editor about the status.

To publish an RN, follow these steps:

  1. Open the main article for the product.

  2. Insert the new article as a component to the main article on top of the previous release articles.

  3. Save the main article.

  4. Click the context menu for the publication and select Publish.

  5. Select the Saved settings tab.

  6. Select the saved settings (there is only one).

    The Zendesk Preflight tab becomes available.

  7. Select the Zendesk Preflight tab, ensure the indicated publication is the correct one and click Publish document.

    The article is published to the HC.

  8. Due to ZD strangeness, re-ordering the articles may be required. Use the Arrange content functionality in the HC admin to drag and drop articles to their correct locations.

To trigger notifications, follow these steps:

  1. Open the article in the HC and click Edit article.

    The editing page opens.

  2. Click Article settings.

    The Article Settings panel opens.

  3. Select the Open for comments option and click Save.

  4. Deselect the Open for comments option and click Save.

  5. Click Close. The Article Settings pane closes.

  6. Select Unpublish from the dropdown menu.

  7. Select Publish Now from the dropdown menu.

    Notifications are sent to users following the section. Huraah.
