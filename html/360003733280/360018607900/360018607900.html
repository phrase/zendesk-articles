v21.44 - August 31, 2021

New Features and Improvements

  • Pre-loaded CAT results now follow the After confirming a segment, jump to settings and any segment filtering. Display of matches within segments has been sped up accordingly.

  • UI has been improved with a checkbox being added for ignoring QA warnings.

Fixed Bugs

  • There was incorrect cursor movement when navigating through segments using the Up and Down arrow keys. This has been fixed.

  • The horizontal scroll bar missing from in-context preview when containing tables. This has been fixed.

  • The Context note tab switched to Preview when clicking on a new segment. This has been fixed.

v21.41 - August 18, 2021

New Features and Improvements

  • LQA issues can now be created on joined jobs.

  • New filter allows segments to be filtered based on Contains tags or Only tags.

  • Segments can now be sorted based on the match rate.

  • Undo/redo operations can now be performed on text within comments and LQA tabs.

Fixed Bugs

  • Custom tags coming from TM or pre-populated text are not being propagated to repetitions. This has been fixed.

  • In certain filtering scenarios, filtered segments were displayed twice. This has been fixed.

  • False QA warnings reported of Empty tag and Inconsistent tag in XML files. This has been fixed.

Fixed Bugs

  • Source preview is not correctly preserved after saving a comment/LQA. This has been fixed.

v21.15.3 - April 28, 2021

Fixed Bugs

  • Custom tags are not propagated in repetitions if there are no tags in source. This has been fixed.

v21.12.6 - April 13, 2021

Fixed Bugs

  • Safari: Double clicking a word does not select it in Mojave version 10.14.5, Safari 12.1.1. This has been fixed.

  • Scrolling stops when repeatedly pressing the PgUp/PgDn keys. This has been fixed.

  • Incorrect apostrophe marks applied for Catalan. This has been fixed.

  • Imported VTT files with newline in a segment are showing as merged. This has been fixed.

  • Entering a double byte character (Japanese) just after a tag (both opening tag of tag pair and single tag ), the first following character is deleted. This has been fixed.

  • Certain characters are displayed in a different font than usual in Simplified Chinese. This has been fixed.

  • A partial MXLIFF without any <m:tunit-target-metadata/> is being sent when tags were deleted from a target when it should be empty. This has been fixed.

  • Line breaks from a term's Note field in a term base are not being preserved. This has been fixed.

  • Joined jobs in RTL languages are presenting segments in the wrong order. This has been fixed.
