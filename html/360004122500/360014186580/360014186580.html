Slow response time from Memsource can be caused either by problems on the Memsource side or by an internet connection.

If experiencing slow response times, follow these steps:

  1. Check the Memsource Status page to see if there are any incidents reported and to see the Web App - Response Time. If the response time is less than 500 ms, Memsource is working correctly. If the response time is higher, contact the Technical Support team.

  2. Verify access to Memsource servers with Host Tracker.

  3. In many cases, the problem is caused by an Internet Service Provider re-routing the connection to Memsource servers via too many other servers, generating slow response times.

    If working with Windows, download an application for tracking connection routes such as WinMTR or similar. Generate a log and discuss it with the provider.
