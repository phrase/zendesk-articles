Style:

We must consider that our customers are not all native English speakers or even English speakers at all. Keeping our HC correct and concise will help those customers who may be using machine translation (Google) to get help with our product. Vaguely worded statements result in support calls.

  • Stick to imperative sentences whenever possible.

    • Avoid conditionals such as would, should, could. They make it harder to write, harder to understand and harder to translate.

    • Avoid addressing the reader with use of you.

      Examples:

      Download the file instead of download your file.

      The editor will instead of Your editor will.

    • Avoid contractions. They may not be understood and/or may not translate.

      Example:

      Application instead of app.

Capitalization:

  • Article titles

    Capitalize nouns and verbs.

  • File extensions

    Capitalize file extensions excepting when given in examples.

  • User Roles

    When referring to a Memsource definition of a user role (Linguist, Project Manager, etc.) capitalize the word. This use should also be linked to the HC article for that role for the first use in a topic.

    If referring to general use of the word, do not capitalize.

    To figure out the difference, consider if a link to the HC article for that use makes sense.

  • UI Reference

    Always follow what is currently in the UI.

  • Memsource specific terms

    Similar to user roles. If referring to a term when specific to Memsource, capitalize it and apply a reference link.

Text formatting:

Avoid use of single quotation marks.

  • Paligo attribute guilabel ALT+G is applied to active elements in the UI such as buttons and menu items. If clicked on, it is an active element.

    guilabel is rendered in the HC as bold.

  • Paligo element guimenu is applied to text used in the UI, but not active such as an option label. If not clicked on, it is a not active.

    guimenu is rendered in the HC as italic.

  • Bold can be applied to headings that are not required for navigation. Use the Bridgehead element on headings that are required for navigation and set the renderas attribute as sect1, sect2, sect3 as required.

  • Emphasis (Italic) is applied to file names and URLs that are not linked.
